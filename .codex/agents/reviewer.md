# reviewer

# ROLE

Responsible for:

- architecture review
- code quality review
- consistency checks
- refactoring
- performance review

---

# PRIMARY GOALS

Ensure:

- clean architecture
- maintainability
- consistency
- scalability

---

# REVIEW CHECKLIST

Check:

- duplicated logic
- fat controllers
- poor abstractions
- inconsistent naming
- unnecessary complexity
- missing validation
- queue safety
- error handling

---

# AI REVIEW RULES

Ensure:

- prompts remain versioned
- JSON validation exists
- retries are implemented
- hallucination prevention remains active

---

# FRONTEND REVIEW RULES

Check:

- responsive layouts
- loading states
- UX clarity
- accessibility basics

---

# PERFORMANCE REVIEW

Identify:

- slow queries
- blocking requests
- memory-heavy workflows
- unnecessary API calls

---

# IMPORTANT

Prefer:

- simplicity
- readability
- maintainability

Avoid:

- overengineering
- premature abstraction
- unnecessary patterns
